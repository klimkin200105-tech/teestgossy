# Тренажёр тестов на Streamlit

Проект содержит два независимых модуля:

1. `pdf_to_excel.py` — локально извлекает вопросы из PDF в Excel.
2. `app.py` — запускает публичный тренажёр тестов на Streamlit.

Файл `main.py` оставлен для совместимости со старым локальным запуском PDF-модуля.

## Как работает публичная версия

Пользователю не нужно устанавливать Python, зависимости или Excel-библиотеки. После публикации на Streamlit Community Cloud пользователь просто открывает ссылку в браузере и решает тесты.

В публичном пользовательском режиме доступны:

- выбор темы;
- тренировка;
- экзамен;
- повтор ошибок;
- скачивание результата экзамена в Excel.

Режим `Редактор вопросов` скрыт от обычных пользователей и доступен только в режиме администратора.

## Файл с вопросами

Приложение берёт вопросы из Excel-файлов в папке `data`.

Файл должен содержать лист `Вопросы` и колонки:

- `Номер вопроса`
- `Текст вопроса`
- `Вариант A`
- `Вариант B`
- `Вариант C`
- `Вариант D`
- `Вариант E`
- `Правильный ответ`
- `Тема`
- `Комментарий`
- `Исходный текст вопроса`
- `Статус проверки`

Для тестирования используются только вопросы, где заполнены:

- текст вопроса;
- хотя бы один вариант ответа;
- правильный ответ.

Правильные ответы приводятся к формату `A`, `B`, `C`, `D`, `E`.

## Режимы приложения

### Режим пользователя

Обычный режим для учеников и слушателей.

Доступны только:

- `Тренировка`;
- `Экзамен`;
- `Повтор ошибок`.

Результаты экзамена не сохраняются на сервере. После завершения экзамена пользователь может скачать Excel-файл с результатом. Файл содержит только данные ответов: дату, режим, номер вопроса, тему, текст вопроса, выбранный ответ, правильный ответ, результат и комментарий.

### Режим администратора

Административный режим нужен для проверки базы вопросов.

Дополнительно доступны:

- временная загрузка Excel-файла через интерфейс;
- `Редактор вопросов`;
- `Статистика`;
- просмотр проблемных вопросов.

Загруженный через интерфейс файл используется только в текущей сессии и не сохраняется постоянно на сервере.

В редакторе исходный файл на сервере не перезаписывается. Отредактированную версию можно скачать отдельным Excel-файлом.

## Локальный запуск для разработки

Если вы запускаете проект на своём компьютере, установите Python 3.10 или новее.

Установите зависимости:

```bash
python -m pip install -r requirements.txt
```

Запустите приложение:

```bash
streamlit run app.py
```

Если команда `streamlit` не найдена, используйте:

```bash
python -m streamlit run app.py
```

На Windows также можно запустить файл `run_app.bat`.

## Публикация на Streamlit Community Cloud

1. Создайте репозиторий на GitHub.
2. Загрузите в репозиторий файлы проекта.
3. Убедитесь, что Excel-файл с вопросами лежит в папке `data`.
4. Убедитесь, что в репозитории есть файлы:
   - `app.py`
   - `requirements.txt`
   - `runtime.txt`
   - `.streamlit/config.toml`
   - папка `data`
5. Откройте https://share.streamlit.io/.
6. Нажмите `New app`.
7. Выберите репозиторий, ветку и файл запуска `app.py`.
8. Нажмите `Deploy`.

Streamlit Community Cloud сам установит зависимости из `requirements.txt`, поэтому пользователям ничего устанавливать не нужно.

## Локальное извлечение PDF в Excel

Этот шаг нужен только администратору базы вопросов.

Положите исходные PDF-файлы в папку `input`.

Запустите:

```bash
python pdf_to_excel.py
```

Готовые Excel-файлы появятся в папке `output`. После ручной проверки перенесите готовый Excel-файл в папку `data`, чтобы публичное приложение использовало именно его.

## Важные замечания

PDF-модуль рассчитан на текстовые PDF-файлы. Если PDF является сканом, текст из него не извлечётся, и потребуется OCR-распознавание.

Перед повторным созданием Excel из PDF закройте соответствующий `.xlsx`-файл в Microsoft Excel, иначе Windows может заблокировать перезапись файла.
